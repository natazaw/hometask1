  • git init - command creates a new Git repository.
  • git status - shows the current state of your Git working directory and staging area
  • git add - adds new or changed files in your working directory to the Git staging area
  • git commit - is used to save your changes to the local repository
  • git log - is a running record of commits
  • git checkout - is used to switch between branches in a repository
  • git branch - is simply a lightweight movable pointer to one of these commits. As you start making commits, you're given a master branch that points to the last commit you made. Every time you commit, the master branch pointer moves forward automatically. Note. The “master” branch in Git is not a special branch.
  • git push - allows you to send (or push) the commits from your local branch in your local Git repository to the remote repository
  • git pull - s used to fetch and download content from a remote repository and immediately update the local repository to match that content
  • git merge is a common practice for developers using version control systems. Merging is a common practice for developers. Whether branches are created for testing, bug fixes, or other reasons, merging commits changes to another branch. It takes the contents of a source branch and integrates it with a target branch
  • git clone - is used to copy an existing Git repository into a new local directory
